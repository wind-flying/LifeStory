# 第一版设计草案

## 目标

做一个本地浏览器可运行的人生小说生成器：

- 玩家前台主要通过选项推进故事
- 后台维护结构化状态，尽量减少吃书
- 默认不开放随时自由写作式输入
- 允许在后期动态获得少量主动干预机会

## 故事结构

- 当前 UI 默认开放：现代都市
- 代码结构预留：多世界配置
- 推进模式：混合式
  - 阶段推进
  - 关键节点展开
- 人生阶段：
  - 幼年
  - 少年
  - 青年
  - 壮年
  - 中晚年与结局

## 强结构状态

核心分层：

- `meta`
- `settings`
- `world_state`
- `protagonist`
- `abilities`
- `relationships`
- `story_memory`
- `intervention_state`

## 提示词职责

- 开局生成
- 回合规划与续写
- 状态更新
- 主动意图解释

当前实现中为了压缩第一版复杂度，把“回合规划”和“正文生成”合并为一次生成，但仍要求模型同步返回结构化状态补丁。

## 世界配置

世界规则不直接写死在提示词里，而是从世界配置层注入。当前已接入：

- `modern_city`

后续新增世界时，优先新增配置，而不是重写整套提示词。

## 主动干预

- 默认 0 次
- 根据成就、关系资本、社会地位、强情感承载动态获得
- 输入的是意图，不是结果命令
- 系统需要尽量顺着玩家诉求，但不能推翻既有事实和世界规则

## 当前已知限制

- 仍依赖模型遵守 JSON 输出约束
- 结构化状态的冲突校验目前较轻
- 还没有多存档列表，只有当前存档
- 还没有对不同模型的兼容性分支
